# **LeetCode 283\. Move Zeroes 題目統整與深度解析**

這是一道在陣列操作（Array Manipulation）中非常經典的題目，經常用於面試中考察對指標操作與空間複雜度的理解。

## **1\. 題目敘述 (Problem Description)**

給定一個整數陣列 nums，請編寫一個函式將所有的 0 移動到陣列的末尾，同時保持非零元素的相對順序。

**限制條件：**

1. 必須在原陣列上進行操作（In-place），不能拷貝額外的陣列。  
2. 盡可能減少操作次數。

**範例：**

* 輸入: nums \= \[0, 1, 0, 3, 12\]  
* 輸出: \[1, 3, 12, 0, 0\]

## **2\. 測驗目的 (Objectives)**

這道題目主要測試開發者以下幾點能力：

* **雙指標技術 (Two Pointers)**：能否有效利用指標在一次遍歷中完成交換或覆蓋。  
* **空間複雜度控制**：要求 $O(1)$ 的額外空間，測試對記憶體管理的敏感度。  
* **邊界條件處理**：當陣列全是 0、沒有 0 或只有一個元素時的處理邏輯。  
* **操作效率優化**：如何在最少次數的寫入操作下完成任務。

## **3\. 思路邏輯 (Logic & Intuition)**

從題目敘述中，我們可以推導出以下邏輯：

### **核心矛盾**

我們需要「移動」非零元素，但又不能建立新陣列。如果直接遇到 0 就刪除並補在最後，會導致頻繁的陣列移動（$O(n^2)$），效率極低。

### **解題關鍵：雙指標法 (Fast & Slow Pointers)**

#### **如何聯想到雙指標？**

當題目出現以下關鍵字時，通常就是**雙指標法**的信號：

1. **In-place (原地操作)**：不能開新陣列，代表我們必須在現有空間進行「覆蓋」或「交換」。  
2. **保持相對順序**：這暗示我們需要一個指標來「記住」目前處理到哪裡，而另一個指標去「尋找」下一個符合條件的元素。  
3. **分類處理**：題目將元素分為「0」與「非 0」，這類將資料分為兩組並重新排列的要求，最適合用指標來區隔邊界。

我們可以設定兩個指標：

1. **慢指標 (slow)**：用來標記「下一個非零元素應該存放的位置」。這可以想像成一個「待填入的坑位」。因此為 **"第一個為零的位置"**  
2. **快指標 (fast)**：用來遍歷整個陣列，尋找非零元素。這可以想像成一個「探路者」。

考慮到如果沒有零，必須要結束搜尋。

**執行步驟：**

* 當 fast 指向的元素不為 0 時，我們將 fast 的值與 slow 的位置進行操作。  
* 如果我們採用「交換」邏輯，那麼 slow 以前的區域都會是非零元素，而 slow 到 fast 之間的區域則會被自動填充為 0。

## **4\. 複雜度分析 (Complexity Analysis)**

* **時間複雜度**: $O(n)$  
  * 我們只需要遍歷陣列一次，其中 $n$ 是陣列的長度。  
* **空間複雜度**: $O(1)$  
  * 只使用了常數個指標變數，沒有使用額外的資料結構，完全符合題目要求。

## **6\. 進階思考 (Optimization)**

在最優解中，我們使用「交換」而非「先覆蓋再補零」，優點在於：

* 如果陣列中完全沒有 0，交換邏輯只會讓 fast 與 slow 同步前進並自體交換，不會有額外的賦值負擔。  
* 如果 0 的數量極多，這種方法能將「寫入操作」最小化。
